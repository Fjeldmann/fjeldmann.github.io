workshop_intro: >
  This workshop introduces a practical way to work with coding agents in modern data and analytics projects.
  It is designed for professionals who already understand data workflows, reporting needs and business logic, but want to use AI more effectively in structured technical work.
  Participants learn how to set up a project an agent can work in, provide the right context, guide work through planning and review, and validate outputs before anything is trusted or deployed.
  During the workshop, we build a small ETL pipeline and a report using SQL and Python through natural language, project instructions and CLI-driven workflows — without needing to write code manually.
  We also cover the professional side of agentic work: permissions, review, secrets, human-in-the-loop, and how to handle code before it reaches production.

workshop_audience:

  - Professionals who know the business problem well, but want to work faster without becoming full-time programmers

workshop_requirements:

  - Familiarity with data workflows, transformation logic or reporting processes is recommended
  - Bring a laptop
  - "Optional: bring your own sample dataset or use the workshop dataset we provide."

workshop_takeaways:
  - A practical understanding of what coding agents can and cannot do in professional data work
  - Experience creating and guiding an agent through a small ETL and reporting workflow
  - A clear mental model of planning, editing and review workflows, and when to use each
  - Hands-on familiarity with project structure, project instructions, AGENTS.md, skills and tool use
  - Better prompting for structured work: defining goals, constraints, acceptance criteria and context
  - A stronger understanding of the human role: shaping the environment, reviewing output and iterating intelligently
  - Practical safety habits for agentic workflows, including permissions, secrets, human approval and production caution
